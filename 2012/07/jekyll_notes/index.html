jekyll笔记

windows下jekyll开发环境

为了便于调试等各种原因，本地的windows系统上安装了jekyll。安装顺序依次为ruby、ruby Devkit、jekyll、rdiscount。因为我不采用Pygments，后面的就忽略咯。

jekyll在windows下的编码

由于windows系统没有采用UTF-8编码，因此需要给jekyll（版本 0.11.2）在打开文件时指定编码，以保证它能够正常工作。

  1. 修改[RubyDIR]\lib\ruby\gems\1.9.1\gems\jekyll-0.11.2\lib\jekyll\convertible.rb 文件27行

     self.content = File.read(File.join(base, name))
 self.content = File.read(File.join(base, name),:encoding=>"utf-8")

  2. 修改[RubyDIR]\lib\ruby\gems\1.9.1\gems\jekyll-0.11.2\lib\jekyll\tags\include.rb 文件23行

     source = File.read(@file)
 source = File.read(@file, :encoding => "utf-8")

jekyll的sitemap生成器

为了方便搜索引擎的收录，sitemap还是必要的。 好在jekyll支持插件，kinnetica编了一个sitemap生成器，下载sitemap_generator.rb丢到_plugins目录，再打开该文件简单修改MY_URL为网站地址即可。

rdiscount windows平台问题 (2013-02-25)

rdiscount版本2.0.7在windows平台下编译会出现错误。github issue。因此只能安装1.6.8版的。

gem install rdiscount -v 1.6.8
