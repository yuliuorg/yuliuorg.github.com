我的Vim（Vim-of-pisheng)

1 Vim初始安装

主要是因为采用了Vundle的插件管理方式，因此一般使用下述步骤。

  • 下载安装
  • 修改.vimrc或\_vimrc文件
  • 安装Git，windows版本可以使用Github Windows，其中带了msysgit，将其bin文件夹添加到Path以便使用git.exe和curl.exe
  • 安装Vundle插件管理软件
  • 通过Vundle安装和管理插件
    Vundle插件名需要区分大小写

2 vim的基本配置

这一部分具体可以参考.vimrc或\_vimrc文件，设置中均进行了注释。

3 Vim插件

  1. Vundle
    vim插件管理包，需要git和curl的支持

  2. FencView
    自动检测文件编码，也可以手动选择文件编码。
    需要iconv.dll的支持，在系统PATH路径下。

  3. VOom
    部分标记语言的标题自动摘要
    需要python 2.X的支持，不能使用64位的。
    由于python的安装包是msi格式的，在win8下如果直接安装会提示缺少dll文件无法安装，实际上是UAC权限问题，但是msi又不没有右键“以管理员身份运行”这个选项，因此需要曲线救国，新建一个bat文件，文件中写入

      msiexec /i msi文件的绝对路径

    即可安装

  4. Markdown
    提供markdown着色功能 为了便于识别两个空格表示的换行，修改~\.vim\bundle\vim-markdown\syntax\markdown.vim，在96行增加

      hi def link markdownLineBreak             Todo

    markdownLineBreak其实在原始vim中已经定义，没有着色而已

  5. The NERD tree
    一个文件浏览器插件，方便在vim中直接切换文件

  6. NERDCommenter
    代码快速注释/取消注释插件

  7. FuzzyFinder
    buffer、文件、命令...的快速切换和浏览器，神器。
    需要L9插件的支持

  8. L9
    vim脚本库。

  9. Simple-Javascript-Indenter
    javascript缩进插件，支持OOP和jQuery

  10. jQuery
    jQuery语法支持，包括关键字，和CSS选择器

  11. neocomplcache
    自动完成插件，不需要ctags，但是可能占用较大的内存。

  12. neocomplcache-snippets_complete
    提供在neocomplcache使用和snipMate.vim相似的功能。

4 Vim的配色

个人在gui下喜欢使用torte的colorscheme，但是原始版本的配色没有考虑Pmenu的样式，使得Pmenu的色调很不搭（背景是粉红)。

打开colors\torte.vim，在文件最后加入下面几行。 hi Pmenu guifg=#c0c0c0 guibg=#404080 hi PmenuSel guifg=#c0c0c0 guibg=#2050d0 hi PmenuSbar guifg=blue guibg=darkgray hi PmenuThumb guifg=#c0c0c0
